## 1. IA (메뉴 구조)

```
로그인 (/login)  ← 역할 선택 포함 (운영자 / 제휴사)
│
홈 대시보드 (/)  ← 역할별 "내 서비스" 카드 + "내 할 일" 섹션
│
├── 1. 파트너사 관리
│   ├── 1-1. 파트너사 목록/조회          /partners        [제휴사: 자사만 표시]
│   └── 1-2. 파트너사 등록/수정          /partners/create | /partners/:id/edit  [운영자 전용]
│
├── 2. 결재 관리 [운영자 전용]           /approvals
│   └── 결재 요청 목록 (대기중 / 처리완료)
│
├── 3. 서비스 관리
│   ├── 3-1. 서비스 홈                   /services        [역할별 서비스 필터]
│   └── 3-2. 개별 서비스 대시보드        /services/:serviceId/*
│       ├── 3-2-1. 정보 수정             /services/:serviceId/info
│       ├── 3-2-2. 알림 관리             /services/:serviceId/notifications
│       ├── 3-2-3. 리워드 관리           /services/:serviceId/rewards
│       ├── 3-2-4. 오퍼 관리             /services/:serviceId/offers
│       └── 3-2-5. 수동 지급             /services/:serviceId/provisions
│
└── 4. 관리자 페이지 [마스터 전용]
    ├── 4-1. 전체 서비스 관리            /admin/services
    ├── 4-2. 사용자 관리                 /admin/users
    └── 4-3. 진입 권한 부여              /admin/users/:id/edit (4-2 수정 내 포함)
```

## 2. 사용자 역할

| 역할 | 설명 | 접근 범위 |
|------|------|-----------|
| **마스터** | 최고 관리자 | 전체 메뉴 접근 (관리자 페이지 포함) |
| **운영자** | 내부팀 서비스 담당자 | 담당 서비스 관리, 결재 승인/반려, 관리자 페이지 접근 불가 |
| **제휴사** | 외부 파트너사 담당자 | 자사 파트너사/서비스 조회, 수정 요청 → 결재 요청으로 처리 |

### 역할별 UI 차이

| 기능 | 운영자 | 제휴사 |
|------|--------|--------|
| 사이드바 > 파트너사 등록/수정 | ✅ 표시 | ❌ 숨김 |
| 사이드바 > 결재 관리 | ✅ 표시 | ❌ 숨김 |
| 파트너사 목록 | 전체 조회 | 자사만 표시 |
| 서비스 목록 | 담당 서비스 | 자사 서비스만 |
| 폼 저장 버튼 | [저장] | [결재 요청] |
| 홈 > 내 할 일 | 처리할 결재 (승인/반려) | 내 요청 내역 |

## 6. 용어 정리

| 용어 | 설명 |
|------|------|
| 파트너사 | 신사업팩토리와 제휴한 외부 회사 |
| 서비스 | 파트너사가 운영하는 개별 앱/서비스 |
| 오퍼 | 외부 API를 등록하고 리워드와 매핑한 단위 |
| 리워드 | 사용자에게 지급하는 보상 (쿠폰/현금/랜덤) |
| 수동 지급 | 운영 목적으로 특정 사용자에게 직접 리워드를 지급하는 기능 |
| 마스터 | 전체 권한을 가진 최고 관리자 계정 |
| 운영자 | 내부팀 담당자. 담당 서비스 관리 + 결재 승인/반려 권한 |
| 제휴사 | 외부 파트너사 담당자. 자사 범위만 조회, 변경 시 결재 요청 필요 |
| 결재 | 제휴사가 신청한 데이터 변경을 운영자가 승인/반려하는 워크플로 |
| CI | 사용자를 식별하는 고유 식별자 (수동 지급 대상 지정 시 사용) |
| Client ID / Secret | OAuth 인증을 위한 서비스 식별 키 쌍 |
