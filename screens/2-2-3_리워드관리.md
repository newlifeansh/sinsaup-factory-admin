# 2-2-3 — 리워드 관리 (v3.0)

> URL: `/services/:serviceId/rewards`
> 접근 역할: **마스터 / 외부팀** (편집) / **제휴사** (조회 전용)
> 최종 업데이트: 2026-04-01

---

## 화면 목표
사용자에게 지급할 보상(리워드)의 종류와 내용을 정의하고, 오퍼와 연결하여 관리합니다. 지급 내역을 조회하고 실패 건에 대한 분석 및 재지급을 수행합니다.

---

## A. 목록 화면

### 헤더 영역
| 요소 | 컴포넌트 | 설명 |
|------|----------|------|
| 페이지 제목 | Text | "리워드 관리" |
| + 리워드 등록 | Button (Primary) | 클릭 시 등록 모달 열림 (마스터/외부팀만 노출) |

### 목록 테이블

| 컬럼명 | 컴포넌트 | 설명 |
|--------|----------|------|
| 리워드 ID | Text | 시스템 자동 생성 고유 ID |
| 리워드명 | Text | 리워드 이름 |
| 유형 | Badge | 쿠폰 / 현금 |
| 연결 오퍼 | Text | 오퍼ID_오퍼명 형식으로 표시 (예: OFR-001_신규가입오퍼) |
| 지급 조건 | Text | 지급 트리거 조건 설명 |
| 상태 | Toggle | 활성/비활성 전환 (마스터/외부팀만 조작 가능) |
| 등록일 | Text | YYYY.MM.DD |
| 관리 | Button | 수정 버튼 → 수정 모달 열림 (마스터/외부팀만 노출) |

> 기존 "오퍼번호" 컬럼은 비노출 → "연결 오퍼(오퍼ID_오퍼명)" 형식으로 변경
> 오퍼번호/상세일련번호/부점코드/텔러번호 등 내부 정보는 리워드 화면에서 절대 노출 금지

---

## B. 등록/수정 모달 (3단계 구조)

### 1단계 — 공통 정보

| 필드명 | 컴포넌트 | 필수 여부 | 설명 |
|--------|----------|-----------|------|
| 리워드명 | Input | 필수 | 리워드 식별 이름 |
| 지급 조건 | Input | 필수 | 리워드가 지급되는 조건 설명 |
| 리워드 유형 | Radio Group | 필수 | 쿠폰 / 현금 중 선택 |
| 연결 오퍼 | Select (Dropdown) | 필수 | 해당 서비스에 등록된 오퍼 목록에서 선택 |

> 연결 오퍼 드롭다운은 해당 서비스(serviceId)에 등록된 오퍼만 표시

### 2단계 — 유형별 상세 (유형에 따라 분기)

#### 쿠폰 선택 시
| 필드명 | 컴포넌트 | 필수 여부 | 설명 |
|--------|----------|-----------|------|
| 쿠폰 ID | Input | 필수 | 쿠폰 식별자 |
| 쿠폰번호 업로드 | Upload | 선택 | CSV 파일 업로드 |
| 지급 안내 문구 | TextArea | 선택 | 사용자에게 표시될 안내 문구 |
| 유효 기간 | RangePicker | 필수 | 쿠폰 유효 시작일 ~ 종료일 |

#### 현금 선택 시
| 필드명 | 컴포넌트 | 필수 여부 | 설명 |
|--------|----------|-----------|------|
| 지급 방식 | Radio Group | 필수 | 고정 / 랜덤 / 제휴사설정 |
| 최대 지급 제한 금액 | InputNumber | 필수 | 1회 지급 시 최대 금액 (원 단위) |

> v3.0 변경: 통장적요 필드 제거, 지급방식 3가지(고정/랜덤/제휴사설정), 최대 지급 제한 금액만 입력
> 기존 현금의 "지급 금액" 및 랜덤의 "최소/최대/당첨확률" 필드 → 지급방식+최대제한금액으로 간소화

### 3단계 — 저장

| 요소 | 컴포넌트 | 동작 |
|------|----------|------|
| 저장 / 수정 | Button (Primary) | 저장 API 호출 후 모달 닫힘, 목록 갱신 |
| 취소 | Button (Default) | 모달 닫힘 |

## C. 지급 내역 조회

### 요약 영역

| 요소 | 컴포넌트 | 설명 |
|------|----------|------|
| 리워드별 요약 카드 | Card Grid | 각 리워드별 성공 건수 / 실패 건수 요약 표시 |
| 상세보기 버튼 | Button | 클릭 시 해당 리워드의 지급 상세 내역으로 이동 |

### 상세 내역 테이블

| 컬럼명 | 설명 |
|--------|------|
| 지급 일시 | YYYY.MM.DD HH:MM:SS |
| 대상 CI | 지급 대상 사용자 CI (마스킹) |
| 리워드명 | 지급된 리워드 이름 |
| 지급 금액 | 실제 지급 금액 |
| 상태 | Badge: 성공(초록) / 실패(빨강) |
| 실패 사유 | 실패 시 사유 표시 |

### 실패 사유 분석

| 실패 사유 분류 | 설명 |
|----------------|------|
| 연동서버장애 | 외부 연동 서버 응답 실패 |
| 계좌지급실패 | 계좌 이체 처리 실패 |
| 미등록고객 | CI에 해당하는 고객 정보 없음 |
| 미분류 | 위 사유에 해당하지 않는 기타 실패 |

### 재지급 프로세스

| 요소 | 컴포넌트 | 설명 |
|------|----------|------|
| 재지급 요청 | Button | 실패 건 선택 후 재지급 요청 (마스터/외부팀만) |
| ITSVC 결재 | Modal | 재지급 사유 입력 → ITSVC 결재 요청 → 승인 후 자동 재지급 |
