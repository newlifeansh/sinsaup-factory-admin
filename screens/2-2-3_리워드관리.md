# 2-2-3 — 리워드 관리

> URL: `/services/:serviceId/rewards`
> 접근 역할: 마스터 / 운영자 (해당 서비스 진입 권한 보유자)

---

## 화면 목표
사용자에게 지급할 보상(리워드)의 종류와 내용을 정의하고 관리합니다.

---

## A. 목록 화면

### 헤더 영역
| 요소 | 컴포넌트 | 설명 |
|------|----------|------|
| 페이지 제목 | Text | "리워드 관리" |
| + 리워드 등록 | Button (Primary) | 클릭 시 등록 모달 열림 |

### 목록 테이블

| 컬럼명 | 컴포넌트 | 설명 |
|--------|----------|------|
| 리워드 ID | Text | 시스템 자동 생성 고유 ID |
| 리워드명 | Text | 리워드 이름 |
| 유형 | Badge | 쿠폰 / 현금 / 랜덤 |
| 지급 조건 | Text | 지급 트리거 조건 설명 |
| 상태 | Toggle | 활성/비활성 전환 |
| 등록일 | Text | YYYY.MM.DD |
| 관리 | Button | 수정 버튼 → 수정 모달 열림 |

---

## B. 등록/수정 모달 (3단계 구조)

### 1단계 — 공통 정보

| 필드명 | 컴포넌트 | 필수 여부 | 설명 |
|--------|----------|-----------|------|
| 리워드명 | Input | 필수 | 리워드 식별 이름 |
| 지급 조건 | Input | 필수 | 리워드가 지급되는 조건 설명 |
| 리워드 유형 | Radio Group | 필수 | 쿠폰 / 현금 / 랜덤 중 선택 |

### 2단계 — 유형별 상세 (유형에 따라 분기)

#### 쿠폰 선택 시
| 필드명 | 컴포넌트 | 필수 여부 | 설명 |
|--------|----------|-----------|------|
| 쿠폰 ID | Input | 필수 | 쿠폰 식별자 |
| 쿠폰번호 업로드 | Upload | 선택 | CSV 파일 업로드 |
| 지급 안내 문구 | TextArea | 선택 | 사용자에게 표시될 안내 문구 |
| 유효 기간 | RangePicker | 필수 | 쿠폰 유효 시작일 ~ 종료일 |

#### 현금 선택 시
| 필드명 | 컴포넌트 | 필수 여부 | 설명 |
|--------|----------|-----------|------|
| 지급 금액 | InputNumber | 필수 | 지급할 현금 금액 (원 단위) |

#### 랜덤 선택 시
| 필드명 | 컴포넌트 | 필수 여부 | 설명 |
|--------|----------|-----------|------|
| 최소 지급 금액 | InputNumber | 필수 | 랜덤 지급 최솟값 (원 단위) |
| 최대 지급 금액 | InputNumber | 필수 | 랜덤 지급 최댓값 (원 단위) |
| 당첨 확률 | Input | 필수 | 당첨 확률 (예: 10%) |

### 3단계 — 저장

| 요소 | 컴포넌트 | 동작 |
|------|----------|------|
| 저장 / 수정 | Button (Primary) | 저장 API 호출 후 모달 닫힘, 목록 갱신 |
| 취소 | Button (Default) | 모달 닫힘 |

---

## 동작 정의

| 케이스 | 동작 |
|--------|------|
| 유형 변경 | 2단계 필드 즉시 교체 |
| 상태 Toggle 변경 | 즉시 API 호출로 상태 변경 |
| 저장 성공 | 모달 닫힘, 목록 갱신, 성공 Toast 표시 |
| 필수 필드 미입력 | 필드 하단 Validation 메시지 표시 |
| 수정 모달 진입 | 기존 리워드 정보 폼에 pre-fill |
