# 3-1 — 전체 서비스 관리 (마스터 전용)

> URL: `/admin/services`
> 접근 역할: **마스터 전용**

---

## 화면 목표
마스터 관리자가 OAuth 인증 플랫폼에 연동될 모든 제휴 서비스를 등록합니다.
인증 정책(OAuth/OIDC) 및 진입 정책 설정은 등록 이후 **개별 서비스 > 정보 수정(2-2-1)** 에서 수행합니다.

---

## A. 목록 화면

### 헤더 영역
| 요소 | 컴포넌트 | 설명 |
|------|----------|------|
| 페이지 제목 | Text | "서비스 관리 (마스터)" |
| + 신규 서비스 등록 | Button (Primary) | 클릭 시 등록 페이지로 이동 |

### 목록 테이블

| 컬럼명 | 설명 |
|--------|------|
| 서비스명 | 서비스 이름 |
| Client ID | OAuth Client ID |
| 파트너사 | 연결된 파트너사명 |
| 상태 | Active / Suspended / Terminated |
| 등록일 | YYYY.MM.DD |
| 관리 | 수정 버튼 → 등록/수정 페이지로 이동 |

---

## B. 신규 서비스 등록 모달

> UI: 목록 화면 우상단 [＋ 신규 서비스 등록] 버튼 → 모달(Modal) 형태로 표시

### 기본 정보 (등록 시 입력)

| 필드명 | 컴포넌트 | 필수 여부 | 설명 |
|--------|----------|-----------|------|
| 서비스명 | Input | 필수 | 서비스 공식 명칭 |
| 파트너사 | Select | 필수 | 연결할 파트너사 선택 |
| KKB 내부 관리번호 | Input | 선택 | 내부 관리용 식별번호 |
| 상태 | Select | 필수 | Active / Suspended / Terminated |

> 등록 완료 시 Client ID가 자동 발급됩니다.
> 인증 정책 및 진입 정책은 등록 후 **개별 서비스 > 정보 수정** 페이지에서 설정합니다. → [2-2-1 정보 수정](./2-2-1_정보수정.md)

---

## 동작 정의

| 케이스 | 동작 |
|--------|------|
| [재발급] 버튼 클릭 | 경고 확인 모달 → API 호출로 Secret 재발급, 화면에 신규 값 표시 (1회성 노출 고려) |
| 탭 이동 | 입력 데이터 유지 |
| 저장 성공 | 목록으로 이동, 성공 Toast 표시 |
| 수정 페이지 진입 | 기존 서비스 정보 탭별 pre-fill |
| 운영자가 URL 직접 접근 | 403 접근 거부 또는 홈으로 리다이렉트 |
