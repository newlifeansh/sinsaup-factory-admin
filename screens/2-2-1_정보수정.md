# 2-2-1 — 정보 수정 (v3.0)

> URL: `/services/:serviceId/info`
> 접근 역할: **마스터 / 외부팀** (편집) / **제휴사** (조회 전용)
> 최종 업데이트: 2026-04-01

---

## 화면 목표
선택된 개별 서비스의 기본 정보, 인증 정책(OAuth/OIDC), 진입 정책, 태그 매니저를 조회하고 수정합니다.

---

## 탭 구조 (4탭)

| 탭 | 설명 |
|----|------|
| 기본 정보 | 서비스명, 파트너사, 유형, 상태, 설명, 대표 운영자 |
| 인증 정책 | Service ID, Secret, Redirect URI, 세션 시간 등 OAuth 설정 |
| 진입 정책 | 이용 가능 연령, 앱 최소 버전 등 진입 조건 |
| 태그 매니저 | 태그 설정 및 관리 |

---

## Tab 1 — 기본 정보

| 필드명 | 컴포넌트 | 필수 여부 | 설명 |
|--------|----------|-----------|------|
| 서비스명 | Input | 필수 | 서비스 공식 명칭 |
| 파트너사 | Input (Readonly) | - | 수정 불가 (전체 서비스 관리에서만 변경) |
| 서비스 유형 | Select | 필수 | 쇼핑 / 구독 / 커머스 / 리워드 |
| 상태 | Select | 필수 | 운영중 / 준비중 / 중단 |
| 서비스 설명 | Textarea | 선택 | 서비스 소개 문구 |
| 대표 운영자 | Input (Readonly) + Button | - | 현재 대표 운영자 표시, [변경] 버튼 클릭 시 ITSVC 결재 프로세스 |

### 대표 운영자 변경

| 요소 | 컴포넌트 | 설명 |
|------|----------|------|
| 대표 운영자 표시 | Input (Readonly) | 현재 대표 운영자명 및 이메일 표시 |
| 변경 버튼 | Button | 클릭 시 ITSVC 결재 요청 모달 표시 |
| ITSVC 결재 모달 | Modal | 변경 대상자 선택 + 사유 입력 → 결재 요청 |

> 대표 운영자 변경은 ITSVC 결재 승인 후 자동 반영

---

## Tab 2 — 인증 정책 (OAuth/OIDC)

| 필드명 | 컴포넌트 | 필수 여부 | 설명 |
|--------|----------|-----------|------|
| Service ID | Input (Readonly) | - | 시스템 자동 발급, 형식: `제휴사영문3자_서비스약자3자_번호` (예: SMP_SHO_001) |
| Client Secret | Input (Readonly, 마스킹) | - | [재발급] 버튼으로 갱신 가능 |
| 서비스 진입 URL | Input (URL) | 필수 | 서비스 메인 진입 URL |
| 허용 Redirect URIs | Input (Text) | 필수 | 복수 입력 가능 |
| 세션 유효시간 (분) | InputNumber | 필수 | 예: 60 |

> - 기존 Client ID → **Service ID**로 명칭 변경, 포맷: `ABC_DEF_001`
> - Service ID는 4-1 전체 서비스 관리에서 최초 등록 시 자동 발급됨
> - [재발급] 클릭 시 확인 모달 → API 호출 → 신규 Secret 1회 노출

---

## Tab 3 — 진입 정책

| 필드명 | 컴포넌트 | 필수 여부 | 설명 |
|--------|----------|-----------|------|
| 이용 가능 연령 | Radio Group | 필수 | 전체 / 성인만 / 청소년 이하 |
| 앱 최소 버전 | Input | 선택 | 예: 3.2.0 |

---

## Tab 4 — 태그 매니저

| 요소 | 컴포넌트 | 설명 |
|------|----------|------|
| 태그 목록 | Table | 등록된 태그 리스트 (태그명, 태그값, 상태) |
| + 태그 추가 | Button | 새 태그 등록 모달 열림 |
| 태그 수정 | Button | 기존 태그 수정 모달 열림 |
| 태그 삭제 | Button (Danger) | 삭제 확인 모달 후 삭제 |

> 태그 매니저는 서비스에 연결된 태그 데이터를 관리하는 탭

---

## 버튼 영역

| 요소 | 컴포넌트 | 동작 |
|------|----------|------|
| 저장 | Button (Primary) | 전체 탭 내용 저장 API 호출 후 Toast 표시 (마스터/외부팀만) |

> 저장 버튼은 페이지 상단 우측에 고정 표시되며, 탭 전환 없이 전체 내용을 한 번에 저장함
> 제휴사 역할일 경우 저장 버튼 숨김 (조회 전용)

---

## 동작 정의

| 케이스 | 동작 |
|--------|------|
| 탭 전환 | 입력 데이터 유지 |
| 저장 성공 | 현재 페이지 유지, 성공 Toast 표시 |
| [재발급] 버튼 클릭 | 확인 모달 → 재발급 API 호출 → 신규 Secret 표시 |
| 대표 운영자 변경 클릭 | ITSVC 결재 요청 모달 표시 |
| 제휴사 접근 | 모든 필드 Readonly, 저장 버튼 숨김 |
| 권한 없는 서비스 URL 직접 접근 | 403 또는 홈으로 리다이렉트 |
