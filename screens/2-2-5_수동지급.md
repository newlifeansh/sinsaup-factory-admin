# 2-2-5 — 수동 지급 (v3.0)

> URL: `/services/:serviceId/provisions`
> 접근 역할: **마스터 / 외부팀 전용** (제휴사 접근 불가 — 사이드바에서 숨김)
> 최종 업데이트: 2026-04-01

---

## 화면 목표
CS, 이벤트 등 운영상의 이유로 특정 사용자에게 리워드를 직접 지급하고 내역을 관리합니다.

---

## 화면 요소

### 헤더 영역
| 요소 | 컴포넌트 | 설명 |
|------|----------|------|
| 페이지 제목 | Text | "수동 지급" |

### 지급 정보 입력 폼

| 필드명 | 컴포넌트 | 필수 여부 | 설명 |
|--------|----------|-----------|------|
| 지급 대상 CI | Input + Button | 필수 | 사용자 CI 직접 입력 또는 [검색] 버튼으로 조회 |
| 지급할 리워드 | Select | 필수 | 해당 서비스에 등록된 리워드 목록에서 선택 |
| 지급 사유 | Input | 필수 | CS 처리, 이벤트 보상 등 사유 입력 |
| 지급 실행 | Button (Primary) | - | 확인 모달 → 지급 API 호출 |

### 지급 내역 테이블

| 컬럼명 | 설명 |
|--------|------|
| 지급 일시 | YYYY.MM.DD HH:MM:SS |
| 지급 대상 (CI) | 지급받은 사용자의 CI (마스킹 처리 고려) |
| 리워드명 | 지급된 리워드 이름 |
| 지급 사유 | 입력된 지급 사유 텍스트 |
| 처리자 | 지급을 실행한 관리자명 |

---

## 동작 정의

| 케이스 | 동작 |
|--------|------|
| CI 검색 버튼 클릭 | 사용자 검색 모달 열림 (또는 인라인 조회) |
| 지급 실행 버튼 클릭 | 확인 모달 표시 ("XX 리워드를 지급하시겠습니까?") |
| 지급 성공 | 성공 Toast 표시, 지급 내역 테이블 즉시 갱신 |
| 존재하지 않는 CI | "해당 사용자를 찾을 수 없습니다" 에러 메시지 |
| 필수 필드 미입력 | 필드 하단 Validation 메시지 표시 |
| 제휴사가 URL 직접 접근 | 403 접근 거부 또는 `/services`로 리다이렉트 |
