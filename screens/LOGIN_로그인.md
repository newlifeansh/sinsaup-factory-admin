# LOGIN — 로그인

> URL: `/login`
> 접근 역할: 전체 (비인증)

---

## 화면 목표
사용자가 이메일과 비밀번호로 시스템에 안전하게 인증하고 로그인합니다.

---

## 화면 요소

| 요소 | 컴포넌트 | 설명 |
|------|----------|------|
| 로고 | Image | 신사업팩토리 로고 |
| 이메일 | Input (type=email) | 이메일 형식 유효성 검사 |
| 비밀번호 | Input (type=password) | 마스킹 처리 |
| 로그인 버튼 | Button (Primary) | 클릭 시 인증 API 호출 → 성공 시 `/` 이동 |

---

## 동작 정의

| 케이스 | 동작 |
|--------|------|
| 로그인 성공 | 홈 대시보드(`/`)로 이동 |
| 이메일/비밀번호 불일치 | 에러 메시지 표시 ("이메일 또는 비밀번호를 확인하세요") |
| 필드 미입력 | 필드 하단 Validation 메시지 표시 |
| 이미 로그인 상태 | `/login` 접근 시 `/`로 리다이렉트 |

---

## 레이아웃 메모
- 중앙 정렬 카드 형태
- 배경: 다크 테마 전체 배경
