# LOGIN — 로그인 (v3.0)

> URL: `/login`
> 접근 역할: 전체 (비인증)
> 최종 업데이트: 2026-04-01

---

## 화면 목표
사용자가 역할을 선택하고, 이메일과 비밀번호로 시스템에 안전하게 인증합니다.

---

## 화면 요소

| 요소 | 컴포넌트 | 설명 |
|------|----------|------|
| 로고 | Image | 신사업팩토리 로고 |
| 역할 선택 | Select (Dropdown) | 마스터 / 외부팀 / 제휴사 중 선택 |
| 이메일 | Input (type=email) | 이메일 형식 유효성 검사 |
| 비밀번호 | Input (type=password) | 마스킹 처리 |
| 로그인 버튼 | Button (Primary) | 클릭 시 인증 API 호출 → 성공 시 `/` 이동 |

---

## 역할 선택 상세

| 역할 | 설명 | 이메일 도메인 |
|------|------|---------------|
| 마스터 | 전체 시스템 관리자 | @kakaobank.com |
| 외부팀 | 카카오뱅크 내부 담당팀 | @kakaobank.com |
| 제휴사 | 외부 파트너사 담당자 | 제한 없음 |

> 역할 선택은 로그인 폼 최상단에 Dropdown으로 배치
> 기본값: 선택 없음 (placeholder: "역할을 선택하세요")

---

## 동작 정의

| 케이스 | 동작 |
|--------|------|
| 역할 미선택 후 로그인 시도 | "역할을 선택하세요" Validation 메시지 표시 |
| 로그인 성공 | 홈 대시보드(`/`)로 이동, 역할에 따라 대시보드 뷰 분기 |
| 이메일/비밀번호 불일치 | 에러 메시지 표시 ("이메일 또는 비밀번호를 확인하세요") |
| 역할-계정 불일치 | 에러 메시지 표시 ("선택한 역할과 계정 정보가 일치하지 않습니다") |
| 필드 미입력 | 필드 하단 Validation 메시지 표시 |
| 이미 로그인 상태 | `/login` 접근 시 `/`로 리다이렉트 |
