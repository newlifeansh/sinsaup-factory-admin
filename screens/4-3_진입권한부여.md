# 4-3 — 진입 권한 부여 (v3.0)

> URL: `/admin/users/:id/edit` (4-2 사용자 수정 화면 내 포함)
> 접근 역할: **마스터 전용**
> 최종 업데이트: 2026-04-01

---

## 화면 목표
마스터 관리자가 '외부팀' 또는 '제휴사' 담당자에게 특정 서비스를 관리할 수 있는 권한을 부여하거나 회수합니다.

---

## 위치
4-2 사용자 관리의 **수정 화면** (`/admin/users/:id/edit`) 내에 권한 설정 영역으로 포함됩니다.

---

## 화면 요소 (수정 화면 내 권한 설정 영역)

### 계정 기본 정보 영역

| 필드명 | 컴포넌트 | 필수 여부 | 설명 |
|--------|----------|-----------|------|
| 이름 | Input | 필수 | 관리자 성명 |
| 이메일 | Input (Readonly) | - | 수정 불가 |
| 역할 | Select | - | 마스터(노랑) / 외부팀(파랑) / 제휴사(초록) Badge 표시 |
| 상태 | Toggle | - | 계정 활성/비활성 |

### 권한 설정 영역

| 요소 | 컴포넌트 | 설명 |
|------|----------|------|
| 레이블 | Text | "담당 서비스" |
| 서비스 목록 | Checkbox.Group | 전체 서비스 목록을 체크박스로 나열 |
| 체크 상태 | pre-fill | 현재 해당 사용자에게 부여된 서비스 체크 표시 |

> - 마스터 역할인 경우 체크박스 비활성화 또는 "전체 접근" 표시로 대체
> - 외부팀 역할인 경우 전체 서비스 중 개별 선택 가능
> - 제휴사 역할인 경우 소속 파트너사의 서비스만 체크박스 활성화

### 버튼 영역

| 요소 | 컴포넌트 | 동작 |
|------|----------|------|
| 저장 | Button (Primary) | 권한 및 계정 정보 저장 API 호출 후 `/admin/users`로 이동 |
| 취소 | Button (Default) | `/admin/users`로 이동 (변경 미저장) |

---

## 동작 정의

| 케이스 | 동작 |
|--------|------|
| 서비스 체크박스 체크 | 해당 서비스 진입 권한 부여 (저장 시 반영) |
| 서비스 체크박스 해제 | 해당 서비스 진입 권한 회수 (저장 시 반영) |
| 저장 성공 | `/admin/users`로 이동, 성공 Toast 표시 |
| 마스터 계정 수정 | 서비스 체크박스 비활성화 처리 (전체 접근) |
| 외부팀 계정 수정 | 전체 서비스 중 선택 가능 |
| 제휴사 계정 수정 | 소속 파트너사의 서비스만 선택 가능 |
| 외부팀/제휴사가 URL 직접 접근 | 403 접근 거부 또는 홈으로 리다이렉트 |
