# 4-1 — 전체 서비스 관리 (v3.0)

> URL: `/admin/services`
> 접근 역할: **마스터 / 외부팀** (마스터=전체, 외부팀=담당 서비스만, 제휴사=메뉴 숨김)
> 최종 업데이트: 2026-04-01

---

## 화면 목표
마스터는 전체 서비스를, 외부팀은 본인 담당 서비스를 조회하고 등록/관리합니다.
인증 정책(OAuth/OIDC) 및 진입 정책 설정은 등록 이후 **개별 서비스 > 정보 수정(2-2-1)** 에서도 수행 가능합니다.

---

## A. 목록 화면

### 헤더 영역
| 요소 | 컴포넌트 | 설명 |
|------|----------|------|
| 페이지 제목 | Text | "전체 서비스 관리" |
| + 신규 서비스 등록 | Button (Primary) | 클릭 시 등록 모달 표시 (마스터만) |

### 목록 테이블

| 컬럼명 | 설명 |
|--------|------|
| 서비스명 | 클릭 시 상세 페이지 진입 |
| Service ID | `제휴사영문3자_서비스약자3자_번호` (예: SMP_SHO_001) |
| 파트너사 | 연결된 파트너사명 |
| 대표 운영자 | 서비스 책임 담당자 |
| 상태 | Active / Suspended / Terminated |
| 출시일 | YYYY.MM.DD |
| 관리 | 수정 버튼 → 상세 페이지로 이동 |

> 기존 Client ID → **Service ID**로 변경 (포맷: `ABC_DEF_001`)
> 기존 목록의 "운영자변경" 버튼 **제거**

### 역할별 목록 범위

| 역할 | 노출 범위 |
|------|-----------|
| 마스터 | 전체 서비스 |
| 외부팀 | 본인 담당 서비스만 |
| 제휴사 | 메뉴 자체 숨김 (접근 불가) |

---

## B. 신규 서비스 등록 모달

> UI: 목록 화면 우상단 [+ 신규 서비스 등록] 버튼 → 모달(Modal) 형태로 표시

### 기본 정보 (등록 시 입력)

| 필드명 | 컴포넌트 | 필수 여부 | 설명 |
|--------|----------|-----------|------|
| 서비스명 | Input | 필수 | 서비스 공식 명칭 |
| 파트너사 | Select | 필수 | 연결할 파트너사 선택 |
| KKB 내부 관리번호 | Input | 선택 | 내부 관리용 식별번호 |
| 상태 | Select | 필수 | Active / Suspended / Terminated |

> 등록 완료 시 Service ID가 자동 발급됩니다 (포맷: `제휴사영문3자_서비스약자3자_번호`)
> 인증 정책 및 진입 정책은 등록 후 **개별 서비스 > 정보 수정** 페이지에서 설정합니다. → [2-2-1 정보 수정](./2-2-1_정보수정.md)

## C. 서비스 상세 페이지 (4탭 구조)

> URL: `/admin/services/:serviceId`

### 탭 구조

| 탭 | 내용 |
|----|------|
| 기본 정보 | 서비스명, Service ID, 파트너사, 대표 운영자, 상태, 출시일 |
| 인증 정책 | Service ID, Client Secret(마스킹), 서비스 진입 URL, 제공 정보(CI/UUID/이름) |
| 진입 정책 | 이용 가능 연령, 앱 최소 버전 |
| 태그 매니저 | 버튼/페이지뷰 태그 활성/비활성 |

### 등록 사용자 관리 (탭 하단 공통 영역)

#### 카카오뱅크 운영자

| 컬럼 | 설명 |
|------|------|
| 이름 | 담당자명 |
| 이메일 | @kakaobank.com |
| 역할 | 마스터 / 외부팀 |
| 상태 | 사용중 |
| 관리 | 삭제 버튼 (confirm 팝업) |

#### 제휴사 담당자

| 컬럼 | 설명 |
|------|------|
| 이름 | 담당자명 |
| 이메일 | 제휴사 이메일 |
| 상태 | **초대중** (주황, 7일 경과 시 자동 삭제) / **사용중** (초록) |
| 관리 | 삭제 버튼 (confirm 팝업 -- 초대중: "초대가 취소됩니다", 사용중: "계정이 삭제됩니다") |

#### 사용자 초대 모달

| 유형 | 동작 |
|------|------|
| 내부 임직원 | 이름/ID 검색 → 선택 → 즉시 등록 |
| 제휴사 담당자 | 이름+이메일 입력 → 초대 메일 발송 → 상태 "초대중" |

## 동작 정의

| 케이스 | 동작 |
|--------|------|
| [재발급] 버튼 클릭 | 경고 확인 모달 → API 호출로 Secret 재발급, 화면에 신규 값 표시 (1회성 노출) |
| 탭 이동 | 입력 데이터 유지 |
| 저장 성공 | 목록으로 이동, 성공 Toast 표시 |
| 수정 페이지 진입 | 기존 서비스 정보 탭별 pre-fill |
| 사용자 초대 성공 | 초대 완료 Toast, 사용자 목록 갱신 |
| 제휴사가 URL 직접 접근 | 403 접근 거부 또는 홈으로 리다이렉트 |
