# 2-1 — 서비스 홈

> URL: `/services`
> 접근 역할: 마스터 / 운영자

---

## 화면 목표
운영자가 자신이 담당하는 서비스 목록을 확인하고, 관리할 서비스를 선택하여 진입합니다.

---

## 화면 요소

### 헤더 영역
| 요소 | 컴포넌트 | 설명 |
|------|----------|------|
| 페이지 제목 | Text | "서비스 관리" |

### 콘텐츠 영역 — 담당 서비스 목록

| 요소 | 컴포넌트 | 설명 |
|------|----------|------|
| 서비스 카드 목록 | Card Grid 또는 List | 담당 서비스를 카드/리스트 형태로 나열 |
| 서비스명 | Text | 서비스 이름 |
| 서비스 로고/아이콘 | Image | 서비스 대표 이미지 |
| 상태 뱃지 | Badge | 운영중 / 준비중 / 비활성 |

---

## 동작 정의

| 케이스 | 동작 |
|--------|------|
| 서비스 카드 클릭 | `/services/:serviceId/info`로 이동 (개별 서비스 대시보드 진입) |
| 담당 서비스 없음 | "담당 서비스가 없습니다. 마스터 관리자에게 권한을 요청하세요." 빈 상태 표시 |
| 마스터 계정 | 전체 서비스 목록 노출 |
| 운영자 계정 | 진입 권한이 부여된 서비스만 노출 |

---

## 역할별 노출 차이

| 역할 | 노출 범위 |
|------|-----------|
| 마스터 | 전체 서비스 |
| 운영자 | 3-3에서 권한 부여된 서비스만 |
